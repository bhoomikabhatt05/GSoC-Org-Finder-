# ✨ About The Project

Finding the right organization for Google Summer of Code can feel overwhelming, especially for students exploring open source for the first time.

GSoC 2026 Org Finder simplifies the process by bringing selected organizations into one searchable, fast, and beginner-friendly platform.

Instead of manually browsing hundreds of organization pages, contributors can:

- 🔍 Search organizations instantly
- 🏷️ Filter by domains and languages
- ⚖️ Compare organizations side-by-side
- 🟢 Discover beginner-friendly issues
- 📈 Analyze repository activity and competition levels
- 💡 Explore project ideas quickly

Designed to help students discover and contribute to open-source organizations more efficiently during GSoC 2026.

# 📚 Features

## 🔍 Smart Organization Discovery

- Full-text organization search
- Domain and category filtering
- Language-based filtering
- Multi-tag filtering
- Competition-level filters
- Smart sorting system

---

## ⚖️ Compare Organizations

Compare up to **3 organizations** side-by-side using:

- ⭐ GitHub stars
- 🍴 Forks
- 🐛 Open issues
- 📅 Activity level
- 📊 Competition level
- 💻 Technologies used

---

## 🟢 Good First Issues Explorer

Dedicated beginner-friendly issue explorer with:

- Live GitHub issue fetching
- Labels and tags
- Comment counts
- Newest-first sorting
- Direct GitHub links

Track real-time repository activity including:

# 🚀 Quick Start

## 1️⃣ Fork The Repository

Click the **Fork** button on GitHub.

---

## 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/GSoC-Org-Finder-.git

cd GSoC-Org-Finder-
```

---

## 3️⃣ Install Dependencies

```bash
npm install
```

---

## 4️⃣ Start Development Server

```bash
npm run dev
```

---

## 5️⃣ Create A Branch

```bash
git checkout -b feature-name
```

---

## 6️⃣ Make Changes

You can contribute by:

- Improving UI
- Adding organizations
- Fixing bugs
- Improving documentation
- Adding new features
- Enhancing filters and search

---

## 7️⃣ Commit Changes

```bash
git add .
git commit -m "Improve README experience"
```

---

## 8️⃣ Push Changes

```bash
git push origin feature-name
```

---

## 9️⃣ Open A Pull Request

Create a pull request and wait for mentor review.
